Background processed

	- fg command can bring the prompt back
	- redirect the background process/Jon stdin/out/err as best practice
Piping and redirection
	- standard data streams -> stdin(0), stdout(1), stderr(2), e.g. cat 1> output.txt or just simply cat > output.txt
		- cat 1>> output.txt 2>> error.txt
		- cat 0< input.txt or just simply cat < input.txt
		- cat 0< input.txt 1> output.txt

		- tty -> identify the location of the current terminal
		- cat 0< input.txt 1> "tty output"

	- piping -> connection std* data streams
		syntax: command1 | command 2 [| command3 ...]
	- redirection
		- syntax: [1,2,3,&]>[|,&] output_file, [1,2,3,&]>> output_file, [0]< input_file
		- &> file/ >& file ==== [1]> file 2>&1
		- &>> file ==== [1]>> file 2>&1

Archiving and compression

	archive -> tar
	compress -> gzip
	1) make a tarball
		tar -cvf tarball_name.tar files[1-n].txt
		tar -tf tarball_name.tar -> test the content of the tarball_name.tar
	2) compress the tarball (gzip, bzip2) -> compression algorithms
		gzip tarball_name.tar	-> GNU zip!
		bzip2 tarball_name.tar
	3) extract
		tar -xvf tarball_name.tar
		gunzip tarball_name.tar.gz
		gunzip -t(table of contents mode) -p(keep the permissions, override umask)
		bunzip2 tarball_name.tar.bz2
	*) in one step
		tar -cvzf tarball_name.tar.gz files[1-n].txt
		tar -cvjf tarball_name.tar.bz2 files[1-n].txt
		tar -xvzf tarball_name.tar.gz

	in case of tar command, to use stdin/out set the filename to dash(-)
